# 10. Project Structure

## Overview

```
my-app/
├── app.cfg                # pas2js config (generated by bv create)
│
├── public/                # Static files (never change)
│   └── index.html         # Base HTML with <div id="app">
│
├── src/                   # SOURCE CODE (work here!)
│   ├── app.bv             # Root component + router config
│   ├── components/        # Reusable components
│   │   ├── Counter.bv
│   │   └── InfoCard.bv
│   └── views/             # Pages (one per route)
│       ├── Home.bv
│       └── About.bv
│
├── generated/             # Generated Pascal (DO NOT EDIT!)
│   ├── main.pas           # Entry point
│   ├── uApp.pas           # app.bv → Pascal
│   ├── uHome.pas          # Home.bv → Pascal
│   └── uCounter.pas       # Counter.bv → Pascal
│
└── dist/                  # Final bundle (for deployment)
    ├── index.html         # Updated with hashes
    └── js/
        ├── rtl.js         # pas2js runtime
        └── main.[hash].js # Compiled App with cache busting
```

## Folder Details

### `src/` - Source Code

**Where you work.** Contains only `.bv` files.

- `app.bv` — Root component. Defines the global layout, app data, and route configuration.
- `views/` — Pages that correspond to routes. Each file generates a tag with the `-page` suffix.
- `components/` — Reusable blocks. Each file generates a tag with its natural name (kebab-case).

### `generated/` - Generated Pascal

**Never edit these files.** They are recreated with every `bv build` or `bv transpile`.

The preprocessor converts each `.bv` into a Pascal unit with:
- Template as a string
- Data as a function
- Methods as procedures
- Style as CSS injection

### `dist/` - Final Bundle (Production)

**For deployment.** Generated via `bv run build`.
- `index.html` — Automatically injects the script with a hash.
- `js/main.[hash].js` — Hashed version to avoid browser cache issues.
- DevTools is automatically removed in this build.

---

## Professional Development Workflow

```bash
# 1. Clean old files (optional)
bv clean

# 2. Start server in a terminal (with network logs)
bv serve

# 3. In another terminal, compile your .bv (with debug mode and app logs)
bv run dev

# 4. Generate optimized production build with hashes
bv run build

# 5. Test the production build locally
bv run preview
```
